# Research: 前週参照表示

## 結論

前週参照は、既存の週選択状態を「参照中の週」として扱い、過去週だけを読み取り専用にする方針で進める。履歴専用の新しい保存モデルは作らず、週ごとの既存データをそのまま再表示する。

## 調査結果

- 既存のアプリ方針は、商品追加画面と購入リスト画面が同じ週状態を共有する構成である。
- 次週が初期表示であるため、過去週はその延長として扱うのが最も自然である。
- 過去週での編集を許可すると、比較・参照の目的と衝突するため、読み取り専用が妥当である。
- 参照対象は直前の週だけでなく、2週間前・3週間前まで必要である。
- 参照できる内容は、購入内容と料理メニューの両方である。

## 採用方針

1. 週状態に「編集可 / 読み取り専用」の区別を持たせる。
2. 過去週を選んだときは追加ボタンと入力導線を無効化する。
3. 表示は同じ一覧を使い、週だけを切り替えて再利用する。
4. 空の過去週はエラーではなく空表示として扱う。

## 保留事項

- どこまで過去週を遡れるかは、現時点では少なくとも 3 週間前までを対象とする。
- 過去週の見た目の強調方法は、実装時に既存 UI と合わせて決める。
