# Feature Specification: 購入リスト画面

**Feature Directory**: specs/003-purchase-list-screen
**Created**: 2026-04-19
**Status**: Draft
**Input**: ユーザー要望: "買い物時に商品の購入漏れを確認する画面。カテゴリ順で表示し、左フリックで購入済みにして一覧から非表示にする。画面下部に削除と元に戻すを表示する。"

## 短い機能名

purchase-list-screen

## Clarifications

- 週の表示範囲: 1週間分の全商品を1画面で確認する（曜日ごとの区切りは表示しない）。
- 購入済みの扱い: 左フリックで `is_purchased=true` とし、一覧から非表示にする。元に戻すは直近の操作をUNDOできるUIで復帰させる。

## User Scenarios & Testing *(mandatory)*

### ユーザーストーリー 1 - 買い物中に購入漏れを確認する (P1)

買い物をする人として、買い物中にカテゴリ順で並んだ1週間分の一覧を見て、購入済み商品を素早く除外して残りを確認したい。

Independent Test: 一覧を開き、複数の商品を左フリックで購入済みにすると、それらが一覧から消え、画面上部の進捗が更新されることを確認する。

Acceptance Scenarios:
1. Given 週次リストがある, When ユーザーが一覧を開く, Then 商品がカテゴリ順で表示される。
2. Given 商品が未購入の状態, When ユーザーが左フリックで購入済みにする, Then 該当項目は一覧から非表示になる。
3. Given ユーザーが複数項目を購入済みにした後, When ユーザーが画面下部の `元に戻す` を押す, Then 最後に購入済みにした項目が元のリストに復帰する。

4. Given メイン画面が表示されている, When ユーザーが画面下部の「購入リスト」タブをタップする, Then 購入リスト画面に遷移し週次リストが表示される。

5. Given 購入リスト画面が表示されている, When ユーザーが画面下部の「商品追加」（＋）をタップする, Then 週単位の商品追加画面が表示され、追加された商品は当該週のリストに反映される。

---

### ユーザーストーリー 2 - 進捗とフィードバック確認 (P2)

買い物をする人として、画面上部で「何件中何件購入済みか」を確認でき、買い忘れの有無を一目で把握したい。

Independent Test: 複数アイテムの購入済み操作を行い、画面上部の "X/Y" 表示が正しく変化することを確認する。

Acceptance Scenarios:
1. Given 初期状態, When ユーザーがn件購入済みにする, Then 進捗表示が更新される。

---

### エッジケース

- すべての商品を購入済みにしたとき、空状態視覚を表示する。
- 元に戻す操作は一段階（直近）を対象とする。また履歴を長く保持する仕様は初期版では対象外とする。
- 同名の商品が複数ある場合はIDで区別する。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-PL-001**: 画面は週次の買い物リストをカテゴリ単位で表示すること。
- **FR-PL-002**: 画面上部に購入進捗（例: 3/20）を表示すること。
- **FR-PL-003**: アイテムの左フリックで `購入済み` にし、一覧から非表示にすること。
- **FR-PL-004**: 画面下部に削除（1件削除）と元に戻すのアクションを常時表示すること。
- **FR-PL-005**: 元に戻すは直近の購入済み操作を復帰させること。
- **FR-PL-006**: 購入済み状態は永続化され、アプリ再起動後も反映されること（ローカル保存）。
- **FR-PL-007**: カテゴリの並び順はユーザーが設定で変更できること（設定画面側の要件）。
- **FR-PL-008**: 購入済みにしたアイテムは `is_purchased` フラグで管理され、UIはこのフラグに基づくこと。

### Non-Functional

- **NFR-PL-001**: オフラインで動作すること（ローカル保存で読み書きできる）。
- **NFR-PL-002**: 一覧表示は応答遅延が小さく、100項目程度でも滑らかにスクロールできること。

## Key Entities

- **WeekList**: 1週間分のリスト（ID, week_start_date, title）
- **WeekListItem**: 個々の買い物項目（ID, week_list_id, item_master_id?, display_name, quantity, unit, category_id, is_purchased, is_deleted, sort_order）
- **Category**: カテゴリ（ID, name, sort_order）

## Success Criteria *(mandatory)*

- **SC-PL-001**: ユーザーが1画面内で週次リストの未購入アイテムを確認し、購買作業が行える。 (観察ベースの検証)
- **SC-PL-002**: 90%以上のテスト参加者が、3分以内に5項目を購入済みにして元に戻す操作を行える。
- **SC-PL-003**: 保存した購入済み状態がアプリ再起動後も保持される。
- **SC-PL-004**: カテゴリ順で並べたとき、カテゴリ間の視認性が維持される（主要なカテゴリが折りたたまれない等）。

## Assumptions

- 初期リリースはローカル保存を前提にする（クラウド同期は後の拡張）。
- Undo は一段階のみを提供する。
- 日ごとの区切りは表示しない（週全体を表示する）。
