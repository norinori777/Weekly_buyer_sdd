# Research: SVG アイコン統一

## 結論

SVG アイコンは、共通ブランド widget を作って主要画面の AppBar に表示する方針で進める。これにより、画像資産の差し替えが 1 箇所に集約され、画面ごとに見た目を揃えられる。

## 調査結果

- 現在のアプリは、購入リスト・商品追加・設定など複数画面で構成されているが、共通のブランド表示はない。
- `assets/weekly_buyer.svg` はすでに配置されているため、実装ではこれを直接使うだけでよい。
- `pubspec.yaml` には PNG の参照が残っているため、SVG へ切り替える際はアセット定義も合わせて更新する必要がある。
- 主要な画面は AppBar を持っているため、そこに小さなブランド icon を加えると、アプリ全体で統一感が出る。

## 採用方針

1. SVG を表示する共通 widget を作る。
2. 主要画面の AppBar に同 widget を使う。
3. PNG 参照を外し、SVG アセットだけを残す。
4. widget テストで表示と回帰を確認する。

## 保留事項

- どの画面までブランド icon を出すかは、主要な AppBar を持つ画面に限定する。
- 既存のタイトル文字は維持し、アイコンは補助的に表示する。
