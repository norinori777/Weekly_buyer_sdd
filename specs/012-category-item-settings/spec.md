# Spec: カテゴリと商品設定（カテゴリ・商品管理）

**短名**: category-item-settings

## 概要
設定画面でカテゴリとカテゴリ配下の商品を管理する機能。カテゴリの追加・編集・削除、商品（マスタ）の追加・編集・削除を可能にする。以下の差分要件を採用する：
- カテゴリは `id`, `name`, `sort_order` のみ扱う（色・説明は不要）。
- 商品は既存データモデルの最小フィールドを利用し、量（quantity）入力はUIから除外する。
- DBスキーマは変更しない（既存テーブルをそのまま利用）。

## 範囲
- カテゴリ：作成・編集（名前のみ）・削除（カテゴリ内に商品がない場合のみ）。
- 商品：作成・編集（名前など既存フィールド）・削除（購入週に含まれている場合は不可）。
- 既存 `WeeklyShoppingRepository` を拡張して整合性チェックを行う。

## ビジネスルール
- BR1: カテゴリ削除は「そのカテゴリに商品が0件」の場合のみ許可する。UIでは削除ボタンを無効化し、理由を表示する。
- BR2: 商品削除は「現在の購入週に含まれていない」場合のみ許可する。参照がある場合は削除を禁止し、現在の週情報を提示する。
- BR3: DBスキーマを変更しないため、削除禁止はリポジトリ層で明示的にチェックして例外を投げる（UIでも事前チェック）。

## データモデル
- 現状のスキーマをそのまま利用する（変更しない）。
- 実装上の扱いのみ明確化：カテゴリは `id,name,sort_order` 相当として扱い、色や説明は保存/編集画面に出さない。

## リポジトリ / API 指定（実装指示）
- 既存 `WeeklyShoppingRepository` に以下の振る舞いを担わせるか、ラッパーを追加する。
  - `Future<void> deleteCategory(int categoryId)`：内部で `countItems(categoryId)` を呼び、0 でなければ `CategoryNotEmptyException` を投げる。
  - `Future<void> deleteItem(int itemId)`：`hasPurchaseEntries(itemId)` を呼び、参照があれば `ItemInPurchaseWeekException` を投げる。
  - 追加/更新は既存の `addCategory`/`updateCategory`/`addItem`/`updateItem` を利用。
  - すべてトランザクション内で実行すること。

## Provider / Notifier（提案）
- `categoriesProvider`（一覧）: AsyncNotifier を使ってカテゴリ一覧を提供。
- `categoryEditorNotifier` : カテゴリの add/update/delete をラップ（UI 用の busy/error 状態を管理）。
- `itemsProvider.family(categoryId)` : 指定カテゴリのマスタ商品一覧を提供。

## UI 変更点
- カテゴリ作成/編集画面: `color` / `description` 入力欄を削除。
- 商品作成/編集画面: `quantity` 入力欄を表示しない（内部的なカラムが残る場合でも編集不可にする）。
- 削除ボタン: 条件により無効化し、無効時は理由ツールチップまたは説明テキストを表示。
- 削除不可時のダイアログには、可能であれば影響のある購入週一覧を示す。

## テスト（必須ケース）
- リポジトリ単体テスト:
  - カテゴリ削除成功（商品0件）
  - カテゴリ削除失敗（商品1件以上）
  - 商品削除成功（購入週に参照なし）
  - 商品削除失敗（購入週に参照あり）
- ウィジェットテスト:
  - カテゴリ作成/編集画面に色・説明欄がないこと
  - 商品作成画面に数量入力欄がないこと
  - 削除ボタンが条件で無効化され、理由が表示されること

## 受け入れ基準
- 設定画面に色/説明/数量の入力欄は存在しない。
- カテゴリは商品0件の場合のみ削除可能で、UI・APIともに阻止される。
- 商品は現在の購入週に含まれている場合に削除できない（UIで理由提示）。
- DBスキーマは変更しない。リポジトリ層での整合性テストが通ること。

## 補足・検討事項
- 削除ポリシー（完全削除 vs 論理削除）：誤操作対策や履歴保全の観点で論理削除（アーカイブ）を検討可能。
- カテゴリ移動（商品のカテゴリ変更）と購入履歴の表示ルール（履歴は移動前のカテゴリ名で残すか）は今後決める必要あり。
